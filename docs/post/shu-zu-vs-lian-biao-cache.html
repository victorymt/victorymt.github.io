数组 vs 链表：CPU Cache 颠覆了教科书

在现代计算机体系中，由于 CPU 的 Cache 机制，数组的效率远远大于链表。

为什么？

数组存储的数据是连续的——加载一个元素时，Cache 会把相邻的元素一起预取进来。遍历数组基本就是顺序访问 Cache，速度极快。

链表存储的数据是分散的——每个节点的地址不连续。遍历链表时，每访问一个节点几乎都是 Cache Miss，CPU 得去内存（甚至主存）里取数据，慢得多。

那插入呢？

常有人说"链表插入快，数组插入慢"。但如果预先不知道要插入在哪里，就需要先遍历找到位置——这一步本身就是 O(n) 的，链表的遍历因为 Cache Miss 更慢。

而对于数组，虽然插入要移动数据，但 CPU 已经优化了连续内存的拷贝操作，实际速度是极其快的。

所以说，很多教科书上的"链表插入快"在现实中已经不成立了——CPU Cache 的出现颠覆了这些传统认知。数组在现代硬件上几乎总是更好的选择。
